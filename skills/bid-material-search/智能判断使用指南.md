## 🎯 设计理念

**让Claude Code来做智能判断，而不是硬编码规则。**

- ❌ 维护成本高，规则越来越多
- ❌ 测试显示成功率只有6%

- ✅ **Claude Code自己查看材料列表，理解语义**
- ✅ **利用LLM的语言理解能力匹配**
- ✅ **代码极简，只保留基础API调用**
- ✅ **自然处理同义词、模糊匹配、上下文理解**
- ✅ **预期成功率：60%+**

## 📦 MaterialHub数据格式说明

### 重要理解

MaterialHub中的材料通常有两种格式，需要不同的处理方式：

#### 格式A：纯图片材料（40%）
- **原始文件**：PNG/JPG图片
- **直接使用**原始图片
- **示例**：营业执照、业绩合同扫描件

```json
{
  "id": 18,
  "title": "营业执照-琪信通达(北京)科技有限公司",
  "current_revision": {
    "files": [
      {
        "file_type": "original",
        "mime_type": "image/png",
        "filename": "17_一、营业执照.png",
        "url": "/api/v2/files/123"
      }
    ]
  }
}
```

#### 格式B：PDF材料（60%）
- **原始文件**：PDF文档
- **MaterialHub会自动提取PDF页面为PNG图片**
- **files字段包含**：
  - `file_type="original"` + `mime_type="application/pdf"` - 原始PDF
  - `file_type="extracted_page"` - 提取的页面图片（每页一个PNG）
- **示例**：身份证（正反面打包成PDF）、人员证书、社保证明

```json
{
  "id": 3,
  "title": "法定代表人身份证明-王春红",
  "current_revision": {
    "files": [
      {
        "file_type": "original",
        "mime_type": "application/pdf",
        "filename": "02_一、法定代表人身份证明.pdf"
      },
      {
        "file_type": "extracted_page",
        "mime_type": "image/png",
        "filename": "page_1.png",
        "url": "/api/v2/files/456"
      },
      {
        "file_type": "extracted_page",
        "mime_type": "image/png",
        "filename": "page_2.png",
        "url": "/api/v2/files/457"
      }
    ]
  }
}
```

### 处理策略

```python
# 优先查找原始图片
image_file = None
for f in files:
    if f.get("file_type") == "original" and f.get("mime_type", "").startswith("image/"):
        image_file = f
        break

# 如果没有原始图片，使用PDF提取的第一页
if not image_file:
    extracted_pages = [f for f in files if f.get("file_type") == "extracted_page"]
    if extracted_pages:
        image_file = extracted_pages[0]  # 使用第一页
```

这样就能处理**所有材料格式**！

## 🔧 使用方法

### 方法1：单个占位符替换（推荐）

适用于需要精确控制的场景。

```python
from bid_material_search.simple_replace import (
    get_all_materials_sync,
    extract_and_insert_material_sync,
    find_placeholders
)

# Step 1: 获取所有材料列表
materials = get_all_materials_sync(limit=100)

print(f"MaterialHub中共有 {len(materials)} 个材料：")
for mat in materials:
    print(f"  [{mat['id']}] {mat['title']} (类型: {mat['doc_type']['name']})")

# Step 2: 扫描占位符
placeholders = find_placeholders("响应文件")
print(f"\n找到 {len(placeholders)} 个占位符")

# Step 3: 让Claude判断并替换
# 占位符: 【此处插入法定代表人身份证正反面扫描件】
# Claude查看材料列表，判断应该使用ID=3的"法定代表人身份证明-王春红"

result = extract_and_insert_material_sync(
    material_id=3,  # Claude判断的最佳匹配
    placeholder="【此处插入法定代表人身份证正反面扫描件】",
    target_file="响应文件/08-授权委托书.md",
    output_dir="响应文件",
    project_name="人工智能应用区级统筹服务项目"
)

if result["success"]:
    print(f"✓ 成功: {result['message']}")
    print(f"  图片: {result['image_path']}")
else:
    print(f"✗ 失败: {result['message']}")
```

## 💡 Claude判断示例

### 示例1：同义词匹配

**占位符**：`【此处插入委托代理人身份证正反面扫描件】`

**MaterialHub材料**：
- [3] 法定代表人身份证明-王春红
- [4] 委托人身份证明-周杨

**Claude判断**：
- "委托代理人" ≈ "委托人"（语义相同，只是简写）
- 选择 material_id = 4

### 示例2：模糊匹配

**占位符**：`【此处插入ISO 27001认证证书扫描件】`

**MaterialHub材料**：
- [13] ISO/IEC 27001 信息安全管理体系认证-琪信通达

**Claude判断**：
- "ISO 27001认证证书" ≈ "ISO/IEC 27001 信息安全管理体系认证"
- 核心标识"27001"一致，语义相同
- 选择 material_id = 13

### 示例3：数字特征码匹配

**占位符**：`【此处插入ISO 9001认证证书扫描件】`

**MaterialHub材料**：
- [22] 高新技术企业证书-琪信通达

**Claude判断**：
- "9001"是ISO 9001质量管理体系的特征码
- 材料中没有包含"9001"的证书
- 高新技术企业证书不是ISO认证
- 判断为：无匹配材料

### 示例4：上下文理解

**占位符**：`【此处插入社保证明扫描件1】`
**文件**：16-企业人员证书及社保证明.md
**上下文**：
```
### 项目经理：【待补充姓名】
- 证书：信息系统项目管理师（高级）
- 社保：【此处插入社保证明扫描件1】
```

**MaterialHub材料**：
- [5] 关键岗位人员社保证明
- [6] 项目经理-纪泽丰

**Claude判断**：
- 上下文是"项目经理"相关
- 占位符描述不明确（"社保证明扫描件1"没有人名）
- 但结合上下文，这是项目经理的社保
- 材料[6]是"项目经理-纪泽丰"
- 选择 material_id = 6（假设其中包含社保证明）

## 🎓 最佳实践

### DO ✅

1. **让Claude查看完整材料列表**
   ```python
   materials = get_all_materials_sync()
   # 显示材料列表给Claude
   for m in materials:
       print(f"[{m['id']}] {m['title']}")
   ```

2. **提供上下文信息**
   ```python
   placeholders = find_placeholders("响应文件")
   # placeholders包含上下文
   for ph in placeholders:
       print(f"占位符: {ph['placeholder']}")
       print(f"上下文: {ph['context']}")
   ```

3. **一次处理一个占位符**
   - 让Claude有时间判断
   - 可以即时反馈错误
   - 用户可以确认每个匹配

4. **处理PDF材料**
   - 检查是否有extracted_page
   - 使用第一页作为代表性图片
   - 添加打印信息说明使用了PDF提取

### DON'T ❌

1. **不要硬编码匹配规则**
   ```python
   # ❌ 不要这样
   if "身份证" in placeholder:
       if "法定代表人" in placeholder:
           material_id = 3
   ```

2. **不要使用复杂的字符串处理**
   ```python
   # ❌ 不要这样
   query = placeholder.replace("【此处插入", "").replace("扫描件】", "")
   query = query.replace("正反面", "").replace("复印件", "")
   ```

3. **不要自动批量替换所有占位符**
   - 让Claude逐个判断
   - 确保每个匹配都正确

4. **不要忽略PDF格式材料**
   - 60%的材料是PDF格式
   - 必须使用extracted_page

## 🔍 故障排查

### 问题1：找不到合适的材料

**原因**：MaterialHub中真的没有该材料

**解决**：
1. 检查材料列表：`get_all_materials_sync()`
2. 确认材料名称和类型
3. 如果确实没有，需要上传到MaterialHub

### 问题2：下载图片失败

**原因**：PDF材料没有提取页面

**检查**：
```python
detail = get_material_detail_sync(material_id)
files = detail['current_revision']['files']
print(f"文件列表: {[f['file_type'] for f in files]}")
```

**解决**：确认MaterialHub已提取PDF页面

### 问题3：匹配不准确

**原因**：占位符描述不够明确

**解决**：
1. 改进占位符描述（如添加人名、项目名）
2. 提供更多上下文给Claude
3. 手动指定material_id
