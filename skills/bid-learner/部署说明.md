## 🎯 使用场景示例

### 场景1：分析报告遗漏字段

**用户对话**：
```
用户：这次分析报告又遗漏了"交付期"字段，导致商务标缺少这部分内容。
用户：记住这个错误，以后不要再犯了。
```

**Bid-Learner 工作流程**：
1. ✅ 验证上下文（涉及"分析报告"、"商务标" → 投标业务）
2. 🔍 识别问题：分析报告遗漏字段
3. 🎯 定位 Skill：bid-analysis（置信度 85%）
4. 📝 生成 Gotcha：
   ```markdown
   ### 商务条件中的交付期字段易遗漏

   **问题**：分析招标文件时，交付期通常在"供应商须知附表"中，
   但 LLM 可能只关注评分标准表，导致遗漏此字段，后续商务标缺少交付期说明。

   **解决**：分析流程必须包含"商务条件清单"，明确提取：
   - 交付期/交货期
   - 付款方式
   - 质保期/维护期
   - 验收标准

   **示例**：略

   *（学习自实践，注入时间：2026-03-18）*
   ```
5. 💉 注入到 bid-analysis/gotchas.md
6. ✅ 确认："已将经验注入到 bid-analysis/gotchas.md"

### 场景2：质检未发现占位符

**用户对话**：
```
用户：质检通过了，但我发现技术方案里还有 "TODO: 补充实施步骤"，
     bid-assembly 怎么没检测出来？
用户：总结这个教训。
```

**Bid-Learner 工作流程**：
1. ✅ 验证上下文（"质检"、"技术方案" → 投标业务）
2. 🔍 识别问题：占位符检测不完整
3. 🎯 定位 Skill：bid-assembly（置信度 90%）
4. 📝 生成 Gotcha：扩展占位符正则表达式，增加 `TODO:` 模式
5. 💉 注入到 bid-assembly/gotchas.md
6. ✅ 确认

### 场景3：非投标业务（拒绝）

**用户对话**：
```
用户：我的 React 组件报错了，useEffect 依赖项不对。
用户：记住这个错误。
```

**Bid-Learner 工作流程**：
1. ❌ 验证上下文失败：
   - 无投标文件 ❌
   - 无 bid-* skills 调用 ❌
   - 关键词："React"、"useEffect" → 非投标业务 ❌
   - 得分：0/3
2. 🚫 拒绝学习："当前对话不在投标业务上下文中，拒绝学习"

## ⚠️ 注意事项

### 作用域控制
- ✅ **只学习投标业务**：招标、磋商、标书、响应文件、评分、资格、报价
- ❌ **拒绝其他领域**：React、API、Database、测试、部署等

### 触发机制
- 用户必须主动说"记住这个错误"才会触发
- 不会自动学习（避免过度学习）

### 定期维护
- 每周运行 `self_check.py` 检测作用域漂移
- 每月审查 gotchas，删除重复/过时条目
